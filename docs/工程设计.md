# 数学题截图结构化识别项目设计

## 第 1 步：统一项目架构

第一阶段采用“模型适配层 + 规则解析层 + 输出层”的分层设计。

```text
image
  -> PaddleOCRAdapter / MockOCR
  -> FormulaRecognizerAdapter / MockFormula
  -> OpenCVFigureDetector / MockFigure
  -> LayoutSegmenter
  -> QuestionTypeClassifier
  -> Parser(single_choice / fill_blank / solution)
  -> JSONExporter + DebugVisualizer
```

设计原则：

- 模型能力通过适配器注入，便于后续替换 PaddleOCR、UniMERNet、pix2tex。
- 结构化逻辑全部落在 Python 规则层，便于调试与修正规则。
- 每个阶段都保留中间结果，方便输出调试 JSON 和可视化 SVG。

对应代码：

- `src/math_question_ocr/adapters.py`
- `src/math_question_ocr/pipeline.py`
- `src/math_question_ocr/segmenter.py`
- `src/math_question_ocr/visualizer.py`

## 第 2 步：数据结构设计

统一数据结构围绕 `DetectedBlock` 和 `QuestionDocument` 展开。

- `DetectedBlock`：统一表示 OCR 文本块、公式块、图形块。
- `FigureRegion`：图形区域。
- `StemSegment`：题干片段，支持文本、公式、图形混排。
- `OptionItem`：单选题选项。
- `BlankItem`：填空题空位。
- `SubQuestionItem`：解答题小问。
- `QuestionDocument`：最终 JSON 对象。

这样做的好处是：

- 任何模型输出先规整成统一块结构，再进入规则解析。
- `stem_segments` 可以保留多模态信息，不被简单拍平成纯文本。
- 三类题型共用一套总结构，差异只体现在 `options`、`blanks`、`subquestions` 的填充。

对应代码：

- `src/math_question_ocr/schemas.py`

## 第 3 步：题型分类逻辑

题型分类优先用稳定规则：

1. 识别到两个及以上 `A/B/C/D` 选项行，判为单选题。
2. 识别到下划线、空括号、空白占位，判为填空题。
3. 其他默认判为解答题。

第一阶段不追求开放场景泛化，因此优先采用高可解释规则，而不是训练分类模型。

对应代码：

- `src/math_question_ocr/classifier.py`

## 第 4 步：区域切分逻辑

区域切分负责从统一块中切出：

- 题号
- 分值
- 题干
- 公式
- 选项
- 空格
- 小问
- 图形

切分方法：

- 对块按 `y, x` 排序。
- 首行正则识别题号和分值。
- 用选项正则提取 `A/B/C/D`。
- 用空位正则提取填空占位。
- 用小问正则提取 `（1）`、`(2)`、`I`、`II` 等。
- 图形区域由检测器提供并写入 `figures`。

对应代码：

- `src/math_question_ocr/segmenter.py`

## 第 5 步：按题型实现解析器

解析器职责：

- `SingleChoiceParser`：抽取题干和选项。
- `FillBlankParser`：抽取题干和空位。
- `SolutionParser`：抽取题干和小问结构。

各解析器共享同一份切分结果，避免重复做空间规则。

对应代码：

- `src/math_question_ocr/parsers/single_choice.py`
- `src/math_question_ocr/parsers/fill_blank.py`
- `src/math_question_ocr/parsers/solution.py`

## 第 6 步：统一 JSON 输出

输出字段固定为：

- `question_no`
- `score`
- `question_type`
- `has_figure`
- `figures`
- `stem_segments`
- `stem`
- `options`
- `blanks`
- `subquestions`

所有对象都实现 `to_dict()`，最终可直接 `json.dumps()`。

对应代码：

- `src/math_question_ocr/schemas.py`
- `src/math_question_ocr/pipeline.py`

## 第 7 步：调试可视化输出

为了便于调试，第一阶段提供两类调试输出：

1. `debug_layout.json`
   - 保存块、分类结果、切分结果、最终结构化结果
2. `debug_layout.svg`
   - 画出文本块、公式块、图形块、选项、小问等区域框

SVG 采用标准库直接生成，不依赖 GUI。

对应代码：

- `src/math_question_ocr/visualizer.py`

## 运行方式

示例脚本：

```bash
python3 -m examples.demo_pipeline
```

测试：

```bash
python3 -m unittest discover -s tests
```
